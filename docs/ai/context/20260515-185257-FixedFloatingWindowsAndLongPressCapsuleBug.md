# 固定悬浮窗口位置与长按胶囊误显示修复

## 当前记录位置

- 胶囊栏：`x=660`，`y=739`
- 长按提示卡片：`x=680`，`y=766`

## 根因

长按 Right Alt 时，主进程的 `updateFloatingBarVisibility()` 会因为按键按下显示胶囊栏。提示卡片拆成独立窗口后，`shortcut-hint` 显示时没有同步隐藏胶囊栏，也没有阻止提示可见期间再次显示胶囊栏。

胶囊栏显示后继续使用最近一次 `voice-state` 文案，因此出现“没有识别到声音”的单独胶囊弹窗。

## 修改计划

- 将胶囊栏默认位置改为当前记录坐标 `660,739`。
- 将提示卡片默认位置改为当前记录坐标 `680,766`。
- 提示卡片显示时隐藏胶囊栏。
- 提示卡片可见期间，`updateFloatingBarVisibility()` 不显示胶囊栏。
- 移除胶囊栏和提示卡片的拖动能力。
- 移除两个窗口的位置记录文件常量、写入函数和 `move/moved` 监听。
